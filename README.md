# Spring 独立桌宠

Spring 是一个 Windows 独立桌宠，使用 PySide6 播放原始 RGBA PNG 帧。程序不会裁剪、居中、重新抠图或转换生产帧；完整的 192×208 透明画布会被原样缩放显示。

运行时使用从批准的高清动画条带提取出的 384×416（2×）帧。原始
`assets/animations` 保持不变，2× 帧位于 `assets/animations@2x`。

## 功能

- 无边框、透明、始终置顶，不显示任务栏图标
- 清单驱动的持续状态、单次动作和系统动画
- 默认按原建议速度的 60% 播放
- 左键拖动时根据实时方向自动播放向左/向右走
- 右键菜单将持续状态和单次动作分组显示
- 主动退出时完整播放挥手告别动画
- 启动时播放比心出场动画，再恢复上次持续状态
- 长时间保持待机时，约每 8–20 分钟随机播放一次比心彩蛋
- 新增可持久化的“打瞌睡”状态
- 新增可持久化的“听音乐”和“记笔记”状态
- 使用 Windows 前台窗口事件自动识别当前活动应用并切换持续状态
- 前台联动采用 400ms 防抖和 5 秒低频校准，不读取窗口标题
- 支持在右键菜单中关闭联动、查看当前匹配、编辑并热加载应用规则
- 滚轮在 50%–250% 之间缩放，默认 125%
- 右键菜单提供 50%–250% 的七个缩放预设
- 低于 200% 时从 2× 帧高质量缩小，200% 时直接使用原生帧
- Windows 打包版支持当前用户开机自启动
- 保存窗口位置、缩放比例和最后一个循环状态
- 恢复时自动把窗口约束到当前可用显示器
- 当前帧 alpha 蒙版决定鼠标命中区域
- 缩放比例按最终物理像素尺寸计算，避免 Windows DPI 额外放大
- 使用清晰优先的非平滑缩放，获得更扎实的像素观感
- 所有动画以对应的 1× 批准帧作为角色几何标准
- 站立、道具、跑步和跳跃分别使用身体核心锚点，保持人物比例与主体一致，同时保留真实动作轨迹

## 开发运行

```powershell
python -m pip install -r requirements.txt
python main.py
```

## 测试

```powershell
pytest
```

测试包含素材尺寸、RGBA、帧数、跳跃轨迹、左右镜像、清单扩展、拖动方向、告别退出、缩放菜单、自启动、前台应用规则、状态优先级和设置恢复。

## 打包

```powershell
pyinstaller --noconfirm --clean spring_pet.spec
```

打包结果位于 `dist/Spring/`。这是 `onedir` 包，素材由 spec 自动收集，运行时不依赖当前工作目录，也不需要用户另外复制素材。

## 操作

- 左键拖动：移动桌宠，并自动播放对应方向的走路动画
- 右键：打开状态、动作、缩放、自启动和前台应用联动菜单
- 滚轮：缩放桌宠
- 右键菜单 → 退出：挥手后关闭程序

## 前台应用规则

首次运行会创建 `%APPDATA%\SpringPet\foreground_rules.json`。规则只匹配
不区分大小写的进程文件名，并且目标必须是动画清单中的持续状态。未配置或
无法识别的应用不会改变当前状态。规则修改后可通过右键菜单“重新加载应用规则”
立即生效；加载失败时程序继续使用上一次有效规则。

## 增加动画

普通持续状态或单次动作由 `animation_manifest.json` 声明。添加对应的
1×/2× 帧目录，并配置 `role`、`menu_group`、`persist`、`return_to` 和
`menu_order` 后，菜单和控制器会自动识别；拖动、退出等系统动画通过唯一的
`trigger` 声明。
